git pull, fetch, rebase, merge

The short and easy answer is that git pull is simply git fetch followed by git merge.

It is very important to note that git pull will automatically merge whether you like it or not. This could, of course, result in merge conflicts. Let’s say your remote is origin and your branch is master. If you git diff origin/master before pulling, you should have some idea of potential merge conflicts and could prepare your local branch accordingly.

清除Mac Storage, Mac 空间不足

By zgpeace

发表于 2015-12-12

清除Mac Storage, Mac 空间不足

清除前只剩下4G, 120G硬盘Other占用89G，OmniDiskSweeper(官网下载)检查Other情况如下：

发现主要是
1、占用20G ~/Library/Developer/Xcode/DerivedData/
2、占用10G ~/Library/Developer/Xcode/iOS DeviceSupport/
清除1中的全部，2中不需要的模拟器，即可。
rm -rf ~/Library/Developer/Xcode/DerivedData/*

因为清除比较频繁，增加一个Alias比较方便。
alias purgeAllBuilds=’rm -rf ~/Library/Developer/Xcode/DerivedData/*’

Mac添加命令别名

1.切换到用户主目录

2.编辑或新建.bash_profile文件

3.添加别名

命令别名设置语法：

alias [别名]=’[指令名称]’

注意：等号两边均无空格，指令名称中如有空格，需用引号包裹

例如：
alias ll=’ls -l’

4.重载该配置文件

5.查看当前所有别名设置

6.storage满的时候只要在Terminal中运行purgeAllBuilds即可

注：Mac别名设置与Linux大体相同，仅配置文件不同。可参考Linux alias命令

清除后Storage截图，Other占用65G

PS：上面的方便比较快速清理Mac的storage，如果需要更深入清理，请参考下面的博客：Xcode磁盘空间大清理

pod-warning

By zgpeace

发表于 2015-11-06

Cocoapods 0.35.0 运行pod时候，有很多警告，解决方案：删掉所有cocoapods，然后重装，可以解决。

1、看看目前有安装cocoapods相关程序：gem list –local | grep cocoapods

2、删除相关cocoapods

sudo gem uninstall cocoapods -v 0.35.0

sudo gem uninstall cocoapods-core

sudo gem uninstall cocoapods-downloader

sudo gem uninstall cocoapods-plugins

sudo gem uninstall cocoapods-search

sudo gem uninstall cocoapods-trunk

sudo gem uninstall cocoapods-try

查看是否还有没相关的： gem list –local | grep cocoapods

运行 terminal in OS X 10.11 ， default enable rootless ,所以gem install “Operation not permitted - /usr/bin/pod”

sudo gem install cocoapods -v 0.35.0

解决方法1: 命令增加 -n /usr/local/bin
